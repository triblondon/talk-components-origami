Polyfills

How do you avoid having the same polyfills in all your components, and ensure they only load when needed?

Declare in module metadata


{
  "description": "Save for later button",
  "browserFeatures": {
    "required": [
      "postmessage",
      "localstorage"
    ],
    "optional": [
      "webaudio"
    ]
  },
}

Express requirements as Modernizr test names.

Cuts the mustard test


if (Modernizr.classlist && Modernizr.queryselector) {
  Modernizr.load([
    {
      test : Modernizr.fontface,
      nope : ['font-face-polyfill.js', 'font-face-polyfill.css']
    },
    {
      test : Modernizr.websockets && window.JSON,
      nope : 'websockets-polyfill.js',
      complete : function() {
        Modernizr.load('my-application-javascript-bundle.js');
      }
    }
  ]);
}

Fallbacks

How do you show NOSCRIPT content in browsers that 'support' script but fail to meet our criteria?

Some crazy ideas

IE doesn't allow inspection of NOSCRIPT tags, so


<![if gt IE 8]> <noscript> <![endif]>
    <div class="noscript"><img ... /></div>
<![if gt IE 8]> </noscript> <![endif]>

But we could also inspect a comment:


<script>document.write('<' + '!--')</script><noscript>
    <img src="image.jpg">
</noscript -->

What we need is an element that:

  • has been supported since the dawn of time
  • has no useful semantic meaning that might be confusing
  • is rarely used, so that getElementsByTagName can find it efficiently
  • allows its content to be accessed by script

The new <u>!

<u><div style="background:url(...)"></div></u>

Because it makes so much more sense if <u> means 'unsupported'

(But doesn't work in browsers that don't support CSS backgrounds: IE 1-3, Opera < 3)

Asset loading

How does a component know what your URL structure is and where to load its static assets (images, fonts) from?

In the component


var weatherUrl = require('o-assets').resolve('/data/2013/12/monthdata.csv', 'o-weather');
xhr.open("get", weatherUrl);

In the site


var assets = require('o-assets');
assets.setGlobalPathPrefix('/resources');
assets.setModulePaths({
  "o-weather": "weather"
});

Results in /resources/weather/data/2013/12/monthdata.csv

Silencing dependencies

If you install "Tweet" and tweet requires "Grid", how do you avoid bundling the whole grid framework in the output CSS?

  • Modules must use dependencies via mixins, not relying on concrete classes
  • Set $o-MODULENAME-is-silent: false when you want concrete classes
  • Otherwise you only get mixins, functions and variables
  • Build service makes top level modules noisy automatically

Service health

If you have a ton of decentralised web services, how do you track their availability, health and metrics?

  • Scan subnets to discover machines
  • Request with no host header to a discovery endpoint
  • Reply with list of all name-based services hosted on that environment
  • Request /__health and /__metrics to get metrics from those services
  • Repeat every minute
  • Graph the hell out of it
